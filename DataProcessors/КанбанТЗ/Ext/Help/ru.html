ТЗ Канбан

Данная обработка служит для отображения данных по работе создания ТЗ дизайнерами.


Настройки Колонки Регистрация ТЗ Теги

  1. Блок Настройки.
    1. Поле проекты.
      1. Возможно добавление нескольких проектов. При закрытии канбана и при последующем открытии значение данного поля восстанавливается.
    2. Поле Сотрудник.
      1. Если данное поле пустое, то на канбане отображаются все задачи по ТЗ организации. В строке «Информация колонке», в графе «Всего задач» указывается количество карточек в каждой колонке. В данном режиме перемещение карточек невозможно.
      2. Если в данном поле указан дизайнер, то на канбане отображаются карточки с изделиями и композициями, с которыми работает указанный дизайнер. В строке «Информация колонке», в графе «Всего задач» указывается количество задач дизайнера без учета фильтров. Например, если у дизайнера две задачи, одна по проекту 01, а вторая 02 и в поле Проект выбран проект 01, то будет отображена одна карточка, но графе «Всего задач» будет число 2. При заполненным дизайнере возможно перемещение карточек.
    3. Поле Поиск.
      1. Поиск осуществляется среди карточек, которые отображены на экране. Для начала поиска нужно в поле «Поиск» начать ввод текста, который входит в название карточки. Это не обязательно должен быть номер. Примеры текста для поиска: «31.104», «Камин», «Гостин» и т.д.
      2. По мере ввода текста поиска будет сокращать список карточек, которые соответствуют значению поиска
      3. В любой момент можно выбрать нужное наименование из списка поиска.
      4. На канбаме отобразится одна карточка с выбранным наименованием и если заполнен дизайнер ее можно перемещать по канбану.
      5. Для возврата к начальному значению необходимо нажать крестик справого края данного поля.
    4. Вкладка Настройки.
      1. Можно изменить в определенных пределах ширину колонок и размер шрифта и сделать более удобным для себя представление канбана. Данные значения сохраняются при закрытии канбана.
    5. Поле Сортировка.
      1. «Важность» в данный момент находится в раз работке.
      2. «Алфавит» корточки располагаются по алфавиту внутри колонки.
      3. «Сроки» карточки располагаются по крайним датам. Если крайняя дата не заполнена, то карточка располагается на верху колонки.
    6. Колонки
      1. «Ожидание»
        1. В документе «Запуск в работу» при нажатии кнопки «Создать задачи планирования» для изделий и композиций в табличной части документа создаются карточки для канбана ТЗ и размещаются в данной колонке.
        2. Возможно переносить карточку в данную колонку – Да.
        3. Возможно забирать карточку из данной колонки – Да.
      2. «Рекламация»
        1. После переноса файла ТЗ в PDM, если у ведущего конструктора есть требования по доработке данного ТЗ, он не принимает ТЗ, и карточка попадает в данную колонку.
        2. Возможно переносить карточку в данную колонку – Нет.
        3. Возможно забирать карточку из данной колонки – Да.
      3. «В работе»
        1. В данной колонке отображаются изделия и комплекты, с которыми в данный момент работает дизайнер. Только к карточкам в этой колонке возможно прикрепление файла ТЗ. В данной колонке действует ограничение на максимальное количество для одного сотрудника.
        2. Возможно переносить карточку в данную колонку – Да.
        3. Возможно забирать карточку из данной колонки – Да.
      4. «Внешнее ожидание»
        1. Переносятся карточки по изделиям или комплектом требуется внешнее согласование с заказчиком
        2. Возможно переносить карточку в данную колонку – Да.
        3. Возможно забирать карточку из данной колонки – Да.
      5. «Внутреннее ожидание»
        1. Переносятся карточки по изделиям или комплектом требуется согласование внутри организации
        2. Возможно переносить карточку в данную колонку – Да.
        3. Возможно забирать карточку из данной колонки – Да.
      6. «Подтверждение»
        1. После переноса файла ТЗ в PDM карточка из колонки "В работе" автоматически переносится в данную колонку. Если Ведущий конструктор принимает ТЗ, то карточка уходит из канбана, если нет, то карточка возвращается в колонку "Рекламация"
        2. Возможно переносить карточку в данную колонку – Нет.
        3. Возможно забирать карточку из данной колонки – Нет.
      7. «Корзина»
        1. Данная колонка доступна только пользователю с полными правами. При перемещении карточки в данную колонку карточка уходит с канбана без условно.
        2. Возможно переносить карточку в данную колонку – Да.
        3. Возможно забирать карточку из данной колонки – Нет.
  2. Регистрация ТЗ.
    1. Для регистрации файла ТЗ в PDM нужно дважды кликнуть мышкой по карточки, которая расположена в колонке «В работе».
      Открывается окно выбора фала. Файл ТЗ должен иметь наименование как у комплекта (композиции) далее «.ТЗ.pdf» (Пример: 31.105.СУ.Композиция.ТЗ.pdf)
    2. В случае удачного переноса файла в нижней части формы будет сообщение о удачном переносе, в случае не удачного переноса, сообщение о ошибке.
    3. В случае ошибки необходимо сообщить об ошибке администратору системы.
  3. Теги.
    1. Если у карточки зеленый тег, это означает что у карточки есть комментарий
      1. Одинарный клик по карточки отобразит последний комментарий внизу формы
    2. По мере добавления новых тегов раздел будет пополняться. Ожидаются теги "Просрочен" и "Приоритет"

    1. От ведущего конструктора.

      1. Кнопка "Доработка" Если Дизайнер считает, что ПКД сделан Конструктором с ошибками, он нажимает кнопку «Доработка». Задача переходит Ведущему конструктору, если Ведущий конструктор согласен с замечаниями Дизайнера, он передает задачу Конструктору, иначе возвращает ее Дизайнеру.
      2. Кнопка "Корректировка" Изменения поступили от Заказчика. Подтверждения не требуется, задача сразу переходит Конструктору. Если для доработки задачи требуется не большие изменения, то нужно установить флаг «Короткая срочная задача». У конструктора данная задача будет отмечена иконкой в виде часов.
      3. Кнопка "Согласование" ПКД передается Заказчику. Задача находится в исполнении у Менеджера. Если Заказчик согласовывает ПКД, Менеджер нажимает кнопку «Согласовано», и задача переходит на этап РКД. Если у Заказчика есть замечания, то задача возвращается к Дизайнеру и данная задача открывается с другим интерфейсом.

        2. После нажатия кнопки появится окно для ввода ожидаемой даты ответа. Также можно добавить комментарий для обоснования даты ответа.
      4. Кнопка "Запрос" ПКД передается Заказчику. Задача находится в исполнении у Менеджера. Когда Заказчик ответит на запрос, Менеджер нажимает кнопку «Возврат с запроса», и задача возвращается к Дизайнеру.
        1. После нажатия кнопки появится окно для ввода ожидаемой даты ответа. Также можно добавить комментарий для обоснования даты ответа.
        2. Дизайнер может создать чек лист, который поможет Менеджеру получить полную информацию от Заказчика
        3. Для этого требуется нажать кнопку "Создать чек лист"

        4. Далее нажать кнопку "Добавить" и ввести вопрос. Вопросов может быть произвольное количество.
        5. Когда Менеджер получит ответ на поставленный вопрос он может проставить флажок на против вопроса и при возврате задачи Дизайнеру он будет видеть на какие вопросы удалось получить ответы.
      5. Если Дизайнеру не требуется согласование с Заказчиком он может нажать кнопку «Согласовано», и задача перейдет на этап РКД.
    2. Возврат с согласования.

      1. Действия кнопок «Согласование», «Запрос», «Согласовано» и «Создать чек лист» аналогичные действия от Ведущего конструктора.
      2. Действие кнопки "Рекламация" аналогично действию кнопки «Доработка». Это считается грубой ошибкой. Ведется учет рекламаций.
      3. Действие кнопки "Редакция" аналогично действию кнопки «Корректировка». Если в процессе согласования Заказчик решил внести изменение в ТЗ.
      4. Кнопка "Контроль КП" Если в процессе согласования Заказчик решил внести изменение в ТЗ, которые могут изменить стоимость проекта, то задача передается на контроль Финансовому директору.
      5. Финансовый директор может:
        1. Отменить работы по данному комплекту (композиции).
        2. Вернуть задачу Дизайнеру, отменив последние изменения Заказчика.
        3. Вернуть задачу Дизайнеру, приняв изменения Заказчика.
